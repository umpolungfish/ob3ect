ob3ect/auto.py: add ZoomChain — Γ-axis traversal of nested IMASM levels — 2026-06-27

New zoom chain capability. A ZoomChain is a sequence of N full Ob3ectArtifacts spanning
from a fine-grained seed concept (Γ=0) to a coarse-grained target concept (Γ=N-1), with
CLINK morphisms describing how each level composes into the next.

New additions:

  _discover_zoom_levels(seed, target, n_levels, temperature)
    Single LLM call returning the N-level Γ hierarchy and N-1 CLINK morphism descriptions.
    Falls back gracefully if no provider is available.

  ZoomLevel(description, artifact, gamma)
    One level's Ob3ectArtifact plus its Γ index.

  ZoomChain(seed, target, levels, clink_morphisms)
    Full chain. .report() prints the Γ-indexed ladder (TANCH, Frobenius, ΔS, CLINK).
    .save() writes zoom_manifest.json.

  zoom_design(seed, target, n_levels=4, ...)  — async
  zoom(seed, target, **kwargs)                — sync wrapper

CLI usage:

  # expand: Paul Atreides → Dune Novel in 5 levels
  python -m ob3ect.auto "Paul Atreides" --zoom-to "Dune Novel" --zoom-levels 5

  # contract: same args reversed
  python -m ob3ect.auto "Dune Novel" --zoom-to "Paul Atreides" --zoom-levels 5

  # suppress Lean output
  python -m ob3ect.auto "hydrogen atom" --zoom-to "periodic table" --zoom-levels 4 --no-scaffold

Output: digital/zoom_<seed>_<target>/ with one subdirectory per Γ level and a
zoom_manifest.json at the root.

Why: every IMASM sequence is self-similar — the bootstrap runs coherently at any
granularity. Γ is the primitive that indexes zoom level. Each level's TANCH is the
VINIT of the next level up; CLINK is the composition operator between adjacent scales.
The zoom chain is the Γ-axis traversal of the Crystal of Types, expressed as a
runnable sequence of ob3ects.

verify which synthesis routes satisfy μ∘δ=id (Frobenius closure).

IMASM sequence per disconnection:
  VINIT → TANCH → FSPLIT → EVALT(δ=0)/EVALF(δ>0) → FFUSE → IMSCRIB [→ IFIX]

Ch3mpilerOb3ect class:

  --quiet: suppress all output; --detail: full IMASM trace per cut

Add document_lift ob3ect: LLM trace annihilation pipeline

digital/document_lift/document_lift_ob3ect.json — full 7-phase ob3ect for
the domain of stripping AI stylistic watermarks from prose.

Phase 0: semiotic-editorial domain, 5 surface tokens, boundary at the
threshold between statistical average and irreducible authorial register.

Phase 1: 12 opcodes assigned — VINIT=Raw Draft, IMSCRIB=Authorial
Fingerprint, AFWD=Trace Stripping, AREV=Voice Injection, FSPLIT=Contamination
Segmentation, FFUSE=Coherence Reassembly, EVALT=Voice Authenticity Gate,
EVALF=Trace Detection Flag, ENGAGR=Ambiguous Register (B-state for patterns
also present in the author's own prior work), IFIX=Committed Phrasing,
CLINK=Revision Pass Composition, TANCH=Anchored Voice.

Phase 2: Frobenius PASS — FFUSE(FSPLIT(doc)) = doc lifted.
Phase 3: ΔS < 0 — lift is anti-entropic; B-state = active revision.
Phase 4: 14-step bootstrap; 4 failure modes catalogued.
Phase 5: 4-pass scheduler (vocabulary → structure → stance → specificity).
Phase 6: cycle cost = editorial effort; delta_s_verdict = ΔS < 0.

Sources: Wikipedia Signs of AI Writing, Springer Nature Apr 2026, arXiv Apr 2024.
